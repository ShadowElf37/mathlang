[FIXED v0.22.0] BUG-6 zero-arg application of a non-zero-arity function silently
  returned an empty tensor instead of erroring. `f = x -> x+1; f()` yielded `[]`
  (shape [0]) rather than an arity error, because the k==0/n==1 case fell into the
  mapping branch and produced a vacuously-empty result. Decision: error on arity

  apply_val, so compiled bodies are covered too. Zero-arg lambdas `() -> e` (n==0)
  still call fine. Regression tests: GUARDS section in tests.sh (bug6.zeroarg.*).
