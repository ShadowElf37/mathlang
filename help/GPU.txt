[1mGPU { ... }[0m evaluates a block on the GPU compute backend. Captured
variables (scalars and real tensors from the surrounding scope) are uploaded,
the block runs on the device keeping intermediates GPU-resident, and only the
final result is downloaded back.

Build with the gpu feature to enable it:  cargo build --release --features gpu
Without it, a GPU block reports "GPU backend not compiled in".

Supported inside a block:
  - elementwise:  + - * / ^   and comparisons  < > <= >= == !=  (-> 0.0/1.0)
  - unary math:   exp ln log2 log10 sqrt cbrt sin cos tan asin acos atan
                  sinh cosh tanh abs sign floor ceil trunc frac
  - reductions:   sum mean min max   (whole-tensor -> scalar)
  - min/max:      two-arg elementwise form, e.g. min(T, 0)
  - stencils:     shift(T,n,axis)  roll(T,n,axis)
                  ops.lap(T,dx[,bc])   ops.grad(T,dx,axis)   (bc: ops.neumann)
  - construct:    tensor((i,j) -> expr, m, n)   тАФ built on the GPU in one kernel;
                  the body may gather from captured tensors, e.g. T[i,j]
  - loops:        iterate(step, x0, n)   scan(step, x0, n)
  - tuples:       (U, V) state for coupled fields; (A, B)[0] indexes a tuple
  - bindings:     c = A + B; c * 2   (last expression is the result)
  - host helper:  get(cell) reads host-side cell state into the block (resolved
                  up front, before any kernel; it is the ONLY host call allowed)

[1mtensor((i,j) -> expr, m, n)[0m builds a tensor directly on the GPU from an index
lambda тАФ no slow per-cell CPU loop. The body may use the index params, captured
scalars, unary math, and gather from captured tensors (T[i,j]). This is the way
to build large initial conditions: u0 = GPU { tensor(...) }.

[1mscan[0m stacks the whole orbit with TIME AS THE FIRST AXIS: a scalar state gives
[n+1]; a grid state of shape [..] gives [n+1, ..] тАФ a spacetime block ready for
!animate2D in a single download.

[1miterate / scan (residency)[0m keep state on the GPU across all n steps,
so PDE/time-stepping stays on the device with no per-step round-trip. The step
may be an inline lambda, a named function, or a unary builtin (exp).

The state can be a TUPLE of tensors for coupled systems. A multi-argument step
destructures it: iterate((U, V) -> (U', V'), (U0, V0), n) advances both fields
together with a single round-trip (upload (U,V), run all n steps, download),
instead of one trip per field per step.

scan likewise accepts a tuple state: a structured tuple (tensor fields) gives a
tuple of per-field time-stacks, each [n+1, ...]; a flat numeric tuple gives a
[n+1, arity] block тАФ identical to the CPU scan.

Precision is f32 on the GPU; results convert back to f64 on download (~7
significant digits). Sensitive workloads (chaotic maps, long symplectic runs)
will diverge from the f64 CPU result тАФ see docs/CONSIDERATIONS.md section 7.

[1mExamples:[0m
[0m> A=[1,2,3]; B=[10,20,30]; GPU { A + B }
result = [11, 22, 33]
> A=[1,4,9]; GPU { sqrt(A) }
result = [1, 2, 3]
> T=ones(1000,1000); GPU { sum(T) }
result = 1000000
> GPU { iterate(x -> 2*x, 1, 10) }
result = 1024
> u=[1,2,3,4]; GPU { iterate(u -> u*0.5, u, 3) }
result = [0.125, 0.25, 0.375, 0.5]
> N=2000; u0=GPU { tensor((i,j)->1.0*((i-1000)^2+(j-1000)^2<2500), N, N) }  # IC on GPU
> GPU { iterate(u -> u + 0.2*ops.lap(u,1,ops.neumann), u0, 200) }   # 4M-cell heat eq
>
> # Animate a GPU sim тАФ stream one resident step per frame (state in a CPU cell):
> step(u) = u + 0.2*ops.lap(u, 1, ops.neumann)
> st = cell(u0); frame = t -> { w = get(st); set(st, GPU { iterate(step, w, 1) }); w }
> !animate2D frame, 0, 400, 400
> # тАжor precompute the whole [n+1,N,N] spacetime block (time first) and play it:
> !animate2D (GPU { scan(step, u0, 400) })           # see examples/gpu_heat2D.math
>
> # тАжor run a live sim FOREVER (bounded memory + backpressure, close window to stop).
> # Coupled (U,V) state lives in one cell; the multi-arg step runs all n substeps
> # on the device with a single round-trip per frame:
> st = cell((U0, V0))
> step(U,V) = (U + Du*ops.lap(U,1,ops.neumann) - U*V*V + F*(1-U),
>              V + Dv*ops.lap(V,1,ops.neumann) + U*V*V - (F+k)*V)
> frame = _ -> { s = GPU { iterate(step, get(st), 20) }; set(st, s); s[1] }
> !animate2Dforever frame, 30                         # see examples/gpu_grayscott.math
