1. [PARTIAL] bytecode VM — lazy OnceLock compilation, stack-based interpreter, tensor
   indexing (Index), nested lambdas (MakeClosure), non-recursive Def::Func in blocks.

   Still possible to add to the bytecode VM:
   1a. Slices in compiled bodies: T[lo..hi] currently falls back; add a Slice instruction
   1b. Ranges in compiled bodies: lo..hi currently falls back; produces a 1-D tensor
   1c. Tensor literals: (1,2; 3,4) currently falls back; straightforward to compile
   1d. Recursive local functions: needs MakeRecFn or a Cell-based two-pass trick
   1e. Loop instruction: bounded iteration (sum(f, lo, hi) style) compiled to a flat
       loop in the VM — also the only recursion-analogue that is GPU-safe
   1f. Move Instruction enum to src/vm.rs so src/gpu/ can import it without pulling
       in all of eval.rs (deferred until eval.rs is split, see item 9)
   1g. LoadBuffer instruction: GPU-side tensor indexing for captured buffers inside
       GPU lambda bodies (v2 GPU feature, see CONSIDERATIONS.md §15)

2.5. Split eval.rs — it is too large (now 4662 lines); defer until the GPU eval path

5. [from eval 0.x, 2026-05-30, see docs/EVALUATION.md] Make iteration first-class,
   scalable, and collectible — the language's primary use case (time-stepping/PDEs)
   currently has no safe, ergonomic iteration form (recursion overflows ~1500 deep;
   the scalable path is the undocumented `sum(k->{set(c,..);0},1,N)` cell hack).
   5a. iterate(f, x0, n) — apply f n times; flat internal loop, O(1) stack, scales
       to 1e6. The safe replacement for the recursive evolve() idiom. Backs TODO 1e
       (VM Loop instruction) and is GPU-safe. Same special-form shape as sum(f,lo,hi).
   5b. scan(f, x0, n) / iterates(f, x0, n) — return the whole orbit
       [x0, f(x0), ..., f^n(x0)] stacked into a tensor (vector states -> rows,
       shape [n+1, d]). Highest-impact addition; retires the cell+sum trajectory hack.
   5c. cumsum / cumprod / diff — standard array scans; trivial, constantly needed.
   5d. Singleton collection literal: make (x,) a length-1 tensor (Python-style), or
       add vec(x). Removes tensor(k->x,1) boilerplate; fixes recursive-accumulator
       base cases where (x) currently collapses to the scalar x.
   5e. Generalize the stacking family (unification): vstack/hstack/append/concat
       should auto-promote ranks (a 1-D vector becomes a row when stacked onto a 2-D
       matrix) and accept scalars/empty operands. Removes manual unsqueeze/reshape
       glue and makes 5b's vector case fall out for free.
