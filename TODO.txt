--- BUGS FIXED (v0.12) ---

BUG: Closure captured-env deep-clone caused exponential blowup.
  Symptom: calling tensor() (or any function with a lambda in its body) hangs
           when several user functions are defined in the same .math session,
           especially after loading heat.math or similar multi-function files.
  Root cause: Val::Fn stored HashMap<String, Val> for captured env.
              Each lambda evaluated via Expr::Lambda did env.vars.clone(),
              making a full deep copy of the entire env at that moment.
              With n chained function definitions each capturing the previous,
              clone cost was O(2^n).
  Fix: Val::Fn now stores Arc<HashMap<String, Val>>.  Cloning a closure is O(1)
       for the captured env (just increments Arc ref count).  All construction
       sites updated in eval.rs and repl.rs.

FEATURE: m -f <file.math> [expr]  — load a .math file from CLI without entering
  the REPL.  import_file() made pub, main.rs handles -f flag.
  (Previously awkward: the only way to use .math files from CLI was via
   MATHLANG_INIT env var + REPL, which required interactive stdin.)
